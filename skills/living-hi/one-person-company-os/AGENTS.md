# Agent Runtime Guide

本仓库优先服务 OpenClaw。

## 运行原则

- 用户始终是创始人和最终决策者。
- 智能体系统默认使用中文。
- 首次进入必须先出“公司创建草案”，确认后再落地。
- 系统以“回合”推进，不以“周复盘”推进。
- 角色默认最小激活，避免第一次就铺满全公司角色。

## 默认智能体

首次确认后，默认只创建：

- `总控台`
- `产品负责人`
- `工程负责人`

按需再创建：

- `设计负责人`
- `增长负责人`
- `用户运营`
- `质量保障`
- `数据分析`
- `财务`
- `法务合规`
- `运维保障`

## 回合上下文

所有角色交接都必须保留这些字段：

- 当前阶段
- 当前回合编号
- 当前回合目标
- 当前状态
- 关键产物
- 当前阻塞
- 下一步最短动作
- 是否需要创始人确认

以 [orchestration/handoff-schema.json](/home/living/.openclaw/workspace/one-person-company/orchestration/handoff-schema.json) 为准。

## 推荐执行顺序

1. 输出公司创建草案
2. 等创始人确认
3. 创建工作区
4. 生成角色智能体 brief
5. 启动首个回合
6. 仅在触发器出现时进入校准
7. 回合完成后再判断是否切换阶段

## 不该做的事

- 不要谎称已经创建了智能体
- 不要在未确认时批量写文件
- 不要默认进入长周期周报模式
- 不要一次性激活过多角色
