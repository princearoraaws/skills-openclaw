# 🌱 儿童成长跟踪与评估技能

## 概述

这个技能帮助家长将零散的孩子成长信息（运动数据、学习表现、老师反馈、日常观察等）结构化记录到 Obsidian 笔记库中，并基于6维发展模型进行趋势分析和培养建议。

孩子的信息可能以各种形式出现：一句话、一段描述、一个数据、一张照片的文字说明、老师的评语截图等。你的工作是理解这些碎片信息的价值，将它们归入正确的维度，写入合适的文档，并在积累足够信息后提供专业的分析洞察。

## 核心文件路径

所有文件位于用户的 Obsidian 笔记库中：

```
g:\笔记\喜悦天地\小喜悦\
├── 喜悦成长跟踪模型.md          ← 6维模型定义（只读参考）
└── 喜悦记录\
    ├── 260401 阶段性评估.md      ← 基线数据（6岁半首次评估）
    ├── 模板-季度评估.md          ← 季度评估模板
    ├── 模板-月度速记.md          ← 月度速记模板
    ├── YYMMDD 阶段性评估.md     ← 实际的季度评估文件
    ├── YYMMDD 月度速记.md       ← 实际的月度速记文件
    └── YYMMDD 事件记录.md       ← 单次事件/里程碑记录
```

## 6 维发展模型

在处理任何输入时，先将信息归类到以下维度：

| 维度 | 代号 | 关注什么 | 典型输入信号 |
|:---|:---|:---|:---|
| 运动与体能 | D1 ⭐ | 跳绳、街舞、新运动、协调性、体能 | "跳绳XX个""街舞升班了""学了游泳" |
| 学业与知识 | D2 | 识字、阅读、数学、学校反馈、粗心 | "考试XX分""会写XX字了""老师说…" |
| 认知与思维 | D3 | 反应速度、工作记忆、专注力、创造力 | "自己想了个办法""能专注XX分钟" |
| 社交与情绪 | D4 | 朋友、冲突、情绪管理、挫折反应 | "跟同学吵架了""交了好朋友""输了就哭" |
| 习惯与自律 | D5 | 作息、自主性、整理、屏幕时间 | "自己收书包了""总要催着写作业" |
| 兴趣与内驱力 | D6 💎 | 主动性、热情、成就感来源 | "他最近迷上了XX""不用叫就去练了" |

D1（运动）是喜悦已识别的天赋主线，记录时给予更高关注度。
D6（内驱力）是所有维度的根基，每次记录都要留意内驱力相关的线索。

## 工作流程

### 流程一：接收信息并记录

当用户提供了关于孩子的信息，按以下步骤处理：

**Step 1 — 理解和分类**

阅读用户输入，判断信息属于哪些维度。一条信息可能跨多个维度。例如"街舞比赛拿了第二名，他有点不开心但第二天又主动要求加练"涉及 D1（运动成绩）、D4（挫折反应）、D6（内驱力）。

**Step 2 — 判断记录类型**

根据信息的性质和时间点，决定写入哪种文件：

| 场景 | 操作 |
|:---|:---|
| 用户明确说"月度记录"或已到月末 | 创建或更新当月的月度速记文件 |
| 用户明确说"季度评估"或已到季末 | 创建季度评估文件，需要引导用户补充各维度信息 |
| 用户分享了一个重要事件/里程碑 | 创建事件记录文件，或追加到当月速记 |
| 用户随口提到一些日常观察 | 追加到当月速记文件，如果该月速记不存在则创建 |
| 用户要求年度复盘 | 汇总该年所有季度评估，生成年度总结 |

**Step 3 — 写入文件**

- 文件命名格式：`YYMMDD 月度速记.md` 或 `YYMMDD 阶段性评估.md` 或 `YYMMDD 事件记录-简短描述.md`
- 日期取当天日期（或用户指定的日期）
- 使用 Obsidian Flavored Markdown（参考 obsidian-markdown 技能的格式规范）
- 如果文件已存在，追加到现有内容的合适位置，不要覆盖

**Step 4 — 即时反馈**

写入后，给用户一个简短的反馈：
- 确认记录了什么
- 标注涉及哪些维度
- 如果发现值得关注的趋势信号（正向🟢或预警🟡🔴），主动指出
- 如果信息中有量化数据（如跳绳次数），与基线或国标做快速对比

### 流程二：月度速记

当用户在月末要求做月度记录，或说"整理一下这个月的情况"时：

1. 读取 `references/template-monthly.md` 获取模板结构
2. 在 `g:\笔记\喜悦天地\小喜悦\喜悦记录\` 创建月度速记文件
3. 如果本月已有零散记录，先汇总已有信息填入模板
4. 对于缺失的维度信息，主动询问用户（但不要一次问太多，分批问）
5. 重点关注"本月惊喜"和"内驱力观察"这两个字段——它们对长期趋势判断最有价值

引导问题示例（自然对话，不要像填表）：
- "这个月有没有什么让你特别惊喜的事？"
- "运动方面最近怎么样？跳绳有测吗？"
- "他最近有特别热衷什么事吗？还是做什么事开始不太愿意了？"

### 流程三：季度评估

这是最重量级的记录，需要覆盖全部 6 个维度。

1. 读取 `references/template-quarterly.md` 获取模板结构
2. 回顾本季度所有月度速记和事件记录，提取已有数据
3. 按6维度逐一与用户确认信息，填充评估表格
4. 对每个维度做趋势判断（🟢正向/🟡关注/🔴干预），给出判断依据
5. 与上一次评估的数据做对比，标出变化方向（↑↓→）
6. 生成综合打分和下季度行动建议

分析时的要求：
- 量化数据必须与基线对比和国标对比（读取 `references/benchmarks.md`）
- 趋势判断要有具体证据支撑，不要只给结论
- 对 D6 内驱力的评估要格外慎重——如果发现内驱力下降的信号，即使其他维度都在进步，也要发出提醒
- 下季度建议要具体可行，不要空泛

### 流程四：临时事件记录

当用户分享了一个重要的单次事件（如比赛、入学第一天、第一次自己做了某件事等），不需要走完整的月度/季度流程，直接：

1. 创建事件记录文件或追加到本月速记
2. 记录事件本身 + 涉及维度 + 你的简短分析
3. 标注这是正面里程碑还是需要关注的信号

### 流程五：趋势分析与建议

当用户问"最近喜悦怎么样""有没有什么需要注意的""整体评估一下"时：

1. 读取最近的所有记录文件（月度速记、季度评估、事件记录）
2. 按6维度汇总趋势
3. 识别：
   - 哪些维度在持续进步
   - 哪些维度出现了变化或波动
   - 是否有预警信号
   - 内驱力的整体走向
4. 生成简洁的趋势分析，重点突出需要行动的建议

## 分析原则

在做任何评估和建议时，遵循以下原则：

### 关于"正常"

6-8岁儿童的个体差异极大，不存在唯一的"正常"标准。当用户问"正常吗"时，给出参考范围而不是绝对判断。把描述性语言（"在同龄人中处于XX水平"）和鼓励性视角结合起来，避免制造焦虑。

### 关于"天赋"

天赋的核心标志是**学习效率**（同样的练习时间达到更高水平）和**自发投入**（不需要外部驱动就愿意持续做），而不是绝对水平的高低。在评估天赋时，区分"练出来的能力"和"天生学得快的能力"。

### 关于建议

- 永远把"保护内在动机"作为最高优先级
- 建议"做什么"的同时也建议"不做什么"
- 避免建议过多课外班——每增加一项都要评估对自由时间和内驱力的影响
- 当数据不足以下结论时，坦诚告诉用户需要更多观察

### 关于情感

家长分享孩子的信息时，背后往往带有期望、焦虑或自豪。你的回应应该：
- 先肯定家长在关注这些事本身就很重要
- 用数据和事实说话，避免空洞的赞美或无根据的担忧
- 当家长表达焦虑时，帮助区分"需要干预的问题"和"正常的发展波动"

## 信号灯系统

每个维度用三色信号灯来标记当前状态，帮助用户快速了解整体态势：

### 🟢 正向信号（继续保持）
表示该维度发展良好，当前策略有效，无需调整。

### 🟡 关注信号（保持观察）
表示该维度出现了一些变化，可能需要微调策略。不代表有问题，但值得在下一次记录时重点关注。

### 🔴 预警信号（需要行动）
表示该维度出现了明显的负面趋势或异常表现，需要主动干预或寻求专业建议。

## 输出格式要求

所有输出文件使用 Obsidian Flavored Markdown。具体规范：

- 使用 YAML frontmatter（tags, date）
- 使用 Obsidian callouts 做重点标注（`> [!note]`、`> [!tip]`、`> [!warning]`）
- 使用 wikilinks 链接到其他记录文件（`[[YYMMDD 阶段性评估]]`）
- 表格用标准 markdown 表格语法
- 标签使用：`喜悦/月度速记`、`喜悦/季度评估`、`喜悦/事件记录`

## 年龄计算

喜悦的出生日期已知信息：在 2026年4月时为6岁半。据此推算出生日期约为 2019年10月。每次记录时根据当前日期自动计算年龄，格式为"X岁X个月"。

## 附加说明

- 如果用户一次性提供了大量信息（如一段语音转写、一段与老师的对话记录），耐心地全部整理——这些零散信息的长期积累价值远大于任何单次记录
- 如果用户提到了跟踪模型中还没有的维度或指标，灵活加入，并建议是否需要更新跟踪模型
- 月度速记追求轻量（5分钟可完成），季度评估追求完整（覆盖6维度+对比+建议）
- 记录语气要温暖但专业，像一个懂儿童发展的朋友在帮忙整理笔记
