# 任务类型与流程管理系统

## 🎯 系统目标
建立自动化的记忆触发机制，在需要时主动创建和提供工作流程。

## 📊 任务复杂度评估标准

### A. 必须建立记忆触发的情况（硬性条件）
1. **步骤数 ≥ 3**：任务包含3个或以上操作步骤
2. **跨系统 ≥ 2**：任务涉及2个或以上系统组件
3. **配置变更**：修改核心配置文件或系统设置
4. **重复出现**：相同或类似任务第2次出现
5. **用户明确要求**：用户指示"记住"、"总结"、"流程"

### B. 复杂度评分体系
```
步骤数：
- 1步：1分
- 2步：2分  
- 3步：4分
- 4+步：6分

涉及系统：
- 单系统：1分
- 双系统：3分
- 多系统：5分

配置影响：
- 无影响：0分
- 局部影响：2分
- 系统影响：4分

总分 ≥ 6分 → 建议建立记忆触发
总分 ≥ 8分 → 必须建立记忆触发
```

## 🏷️ 任务类型标签系统

### 核心标签
- `系统配置`：修改系统设置、配置文件
- `多步骤`：包含多个操作步骤
- `可重复`：未来可能重复执行
- `跨系统`：涉及多个系统组件
- `重要操作`：影响系统稳定性或功能
- `团队协作`：涉及多个Agent或用户协作

### 标签应用示例
1. **飞书机器人配置**：`[系统配置, 多步骤, 可重复, 跨系统, 重要操作]`
2. **文件读取**：`[简单操作, 单步骤]`
3. **模型测试**：`[系统配置, 可重复]`

## 🔄 主动询问机制

### 触发条件
1. **复杂度触发**：任务复杂度评分 ≥ 6分
2. **重复触发**：相同任务类型第2次出现
3. **关键词触发**：检测到"配置"、"设置"、"建立"等动词
4. **用户意图触发**：检测到"记住"、"总结"、"流程"等词

### 询问模板
```
"孙老师，这个【任务类型】任务（复杂度评分：X分）看起来可能会重复使用，需要我创建标准流程文档吗？"
```

## 📁 流程文档管理

### 文档命名规范
- `[任务类型]-配置流程.md`
- 示例：`飞书机器人配置流程.md`

### 文档结构要求
1. **流程概述**：任务目标和价值
2. **配置步骤**：详细的操作步骤
3. **验证方法**：如何验证配置成功
4. **常见问题**：可能的问题和解决方案
5. **相关文件**：涉及的配置文件和文档

```

```

### 触发动作
1. 检测到触发词 → 搜索相关流程文档
2. 找到文档 → 主动提供："需要我回顾【任务类型】流程吗？"
3. 未找到文档 → 询问："需要我创建【任务类型】的流程文档吗？"

## 🔧 系统维护

### 定期检查
- 每周检查任务历史，识别未建立流程的高复杂度任务
- 每月优化复杂度评分标准
- 每季度审查和更新所有流程文档

### 优化机制
- 根据用户反馈调整触发条件
- 根据实际使用情况优化关键词
- 根据任务执行效果改进流程文档

## 📈 效果评估指标

### 量化指标
1. **流程覆盖率**：高复杂度任务中已建立流程的比例
2. **主动询问率**：在需要时主动询问建立流程的比例
3. **用户提醒率**：需要用户提醒才提供流程的比例（目标：降低）
4. **流程使用率**：流程文档被实际引用的频率

### 质量指标
1. **流程完整性**：步骤是否完整、清晰
2. **触发准确性**：关键词触发是否准确
3. **用户满意度**：流程是否真正帮助了工作
