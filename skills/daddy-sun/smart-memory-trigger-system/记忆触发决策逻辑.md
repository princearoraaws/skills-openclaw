# 记忆触发决策逻辑

## 🧠 决策流程图
```
开始任务
    ↓
分析任务描述
    ↓
评估复杂度（步骤数、涉及系统、配置影响）
    ↓
检查任务历史（是否重复类型？）
    ↓
分析用户意图（是否有"记住"、"总结"等词？）
    ↓
    ┌─────────────────────┐
    │ 是否需要建立记忆触发？ │
    └─────────────────────┘
            ↓
    ┌─────────────────────┐
    │ 是 → 主动询问用户     │
    │ 否 → 正常执行任务    │
    └─────────────────────┘
```

## 📊 复杂度评估算法

### 输入：任务描述
### 输出：复杂度评分（0-10分）

```javascript
function 评估复杂度(任务) {
  let 分数 = 0;
  
  // 1. 步骤数评估
  if (任务.包含("第一步") && 任务.包含("第二步")) 分数 += 2;
  if (任务.包含("第三步")) 分数 += 2;
  if (任务.包含("第四步") || 任务.包含("最后")) 分数 += 2;
  
  // 2. 系统涉及评估
  if (任务.包含("配置文件") || 任务.包含(".json")) 分数 += 1;
  if (任务.包含("命令行") || 任务.包含("命令")) 分数 += 1;
  if (任务.包含("重启") || 任务.包含("网关")) 分数 += 1;
  if (任务.包含("飞书") || 任务.包含("微信") || 任务.包含("平台")) 分数 += 1;
  
  // 3. 配置影响评估
  if (任务.包含("配置") || 任务.包含("设置")) 分数 += 1;
  if (任务.包含("修改") || 任务.包含("编辑")) 分数 += 1;
  if (任务.包含("添加") || 任务.包含("删除")) 分数 += 1;
  
  // 4. 协作需求评估
  if (任务.包含("团队") || 任务.包含("协作")) 分数 += 1;
  if (任务.包含("多个") || 任务.包含("几个")) 分数 += 1;
  
  return 分数;
}
```

## 🔍 重复模式检测

### 任务类型识别关键词
```
系统配置类：["配置", "设置", "安装", "部署"]
文件操作类：["创建", "编辑", "删除", "移动"]
命令行类：["执行", "运行", "命令", "脚本"]
平台集成类：["飞书", "微信", "API", "集成"]
```

### 重复检测逻辑
```javascript
function 检测重复模式(当前任务, 任务历史) {
  // 1. 提取当前任务类型关键词
  const 当前关键词 = 提取关键词(当前任务);
  
  // 2. 在历史中搜索相似任务
  for (const 历史任务 of 任务历史) {
    const 相似度 = 计算相似度(当前关键词, 历史任务.关键词);
    if (相似度 > 0.7) {
      return {
        是重复模式: true,
        相似任务: 历史任务,
        相似度: 相似度
      };
    }
  }
  
  return { 是重复模式: false };
}
```

## 🎯 用户意图分析

### 明确要求关键词
```
记录要求：["记住", "记录", "保存", "备忘"]
总结要求：["总结", "归纳", "整理", "梳理"]
流程要求：["流程", "步骤", "方法", "操作"]
未来使用：["下次", "以后", "将来", "重复"]
```

### 意图分析逻辑
```javascript
function 分析用户意图(用户输入) {
  const 意图 = {
    需要记录: false,
    需要总结: false,
    需要流程: false,
    未来使用: false
  };
  
  if (用户输入.包含("记住") || 用户输入.包含("记录")) 意图.需要记录 = true;
  if (用户输入.包含("总结") || 用户输入.包含("归纳")) 意图.需要总结 = true;
  if (用户输入.包含("流程") || 用户输入.包含("步骤")) 意图.需要流程 = true;
  if (用户输入.包含("下次") || 用户输入.包含("以后")) 意图.未来使用 = true;
  
  return 意图;
}
```

## 💬 主动询问决策

### 决策条件（满足任一即可）
1. **复杂度条件**：评分 ≥ 6分
2. **重复条件**：检测到重复模式（相似度 > 0.7）
3. **意图条件**：用户明确要求记录/总结/流程
4. **配置条件**：涉及系统配置文件修改

### 询问模板选择
```
// 基于复杂度
"孙老师，这个任务看起来比较复杂（评分：X分），可能会重复使用，需要我创建流程文档吗？"

// 基于重复模式  
"孙老师，这个任务和之前的【相似任务】很相似，需要我参考之前的流程吗？"

// 基于用户意图
"孙老师，您提到需要【记录/总结/流程】，我现在就创建文档吗？"

// 基于配置重要性
"孙老师，这个配置变更会影响系统稳定性，需要我记录详细步骤吗？"
```

## 📝 流程创建决策

### 必须创建流程的情况
1. 复杂度评分 ≥ 8分
2. 用户明确说"记住这个流程"
3. 涉及核心系统配置（如openclaw.json）
4. 任务已重复出现第2次

### 建议创建流程的情况
1. 复杂度评分 6-7分
2. 用户提到"可能还会用"
3. 涉及多个Agent协作
4. 有潜在的错误风险

### 不需要创建流程的情况
1. 复杂度评分 ≤ 5分
2. 明显的一次性操作
3. 简单的查询或信息获取
4. 探索性的试错操作

## 🔄 执行流程

### 完整决策流程
```
1. 接收任务
2. 评估复杂度（评分）
3. 检查重复模式
4. 分析用户意图
5. 综合决策：
   - 必须创建 → 直接询问"我现在创建流程文档吗？"
   - 建议创建 → 询问"需要我创建流程文档吗？"
   - 不需要 → 正常执行
6. 如果创建流程：
   - 确定任务类型和标签
   - 选择触发关键词
   - 创建流程文档
   - 更新任务历史
```

### 实时调整机制
- 如果用户说"不用记录" → 尊重选择，但记录决策
- 如果用户说"简单的不需要" → 调整复杂度阈值
- 如果用户提供更好方法 → 更新流程文档

## 📈 效果反馈循环

### 收集反馈
1. **用户直接反馈**："这个流程很有用" / "不需要这么复杂"
2. **使用频率**：流程文档被引用的次数
3. **错误减少**：建立流程后同类错误是否减少
4. **效率提升**：任务执行时间是否缩短

### 优化调整
```
收集反馈 → 分析效果 → 调整决策参数 → 改进流程质量
    ↓
更好的记忆触发 → 更高的工作效率 → 更满意的用户体验
```
