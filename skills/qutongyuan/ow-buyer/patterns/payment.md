# 支付平台集成

## 支付平台支持

### 国内支付

| 平台 | 接口类型 | 适用场景 |
|------|----------|----------|
| 支付宝 | 扫码支付/转账 | 国内主流，个人/企业 |
| 微信支付 | 扫码支付/转账 | 国内主流，个人/企业 |
| 银行转账 | 对公转账 | 大额企业采购 |

### 国际支付

| 平台 | 接口类型 | 适用场景 |
|------|----------|----------|
| Apple Pay | 应用内支付 | iOS 用户 |
| PayPal | 账户支付/信用卡 | 国际交易 |
| Stripe | 信用卡支付 | 国际电商 |
| Google Pay | 应用内支付 | Android 用户 |

## 支付流程

### 1. 生成支付订单

确认中标后：
```json
{
  "tx_id": "TX-20260330-001",
  "req_id": "REQ-20260330-001",
  "bid_id": "BID-xxx",
  "buyer": "小恩",
  "supplier": "供应商A",
  "amount": 2800,
  "currency": "CNY",
  "item": "幽灵庄园红酒",
  "status": "pending_payment"
}
```

### 2. 选择支付方式

询问用户：
```
请选择支付方式：
1. 支付宝扫码
2. 微信支付扫码
3. Apple Pay
4. PayPal
5. 银行转账
```

### 3. 生成支付链接

**支付宝：**
```
生成支付宝收款码/转账链接
金额：¥2,800
收款方：供应商A（支付宝账户）
```

**微信支付：**
```
生成微信收款码
金额：¥2,800
收款方：供应商A（微信账户）
```

**PayPal：**
```
生成 PayPal 支付链接
金额：$400 (约 ¥2,800)
收款方：supplierA@example.com
```

### 4. 确认支付

用户完成支付后：
- 记录支付时间、方式、凭证
- 更新交易状态为 `paid`
- 通知供应商发货

## 安全措施

### 支付前确认
1. 验证供应商身份（签名验证）
2. 确认交易金额与投标价格一致
3. 生成交易合同/协议
4. 保留完整投标记录作为交易凭证

### 支付保护
1. 建议使用平台担保交易（如支付宝担保）
2. 大额交易建议银行对公转账
3. 保留支付凭证截图
4. 设置交易超时（未发货自动退款）

### 纠纷处理
1. 记录所有沟通记录
2. 保留商品照片/视频证据
3. 支持申请退款/投诉
4. 记录到供应商信誉档案

## 未来扩展

### 智能合约支付
- 使用区块链智能合约托管资金
- 确认收货后自动释放付款
- 不可篡改的交易记录

### 平台担保
- 类似淘宝担保交易
- 买家确认收货后卖家收款
- 支持退款申请

### 信用支付
- 信誉高的供应商可先发货后付款
- 信誉高的买家可先收货后付款
- 建立信用评级体系
